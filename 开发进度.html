💡 关键概念速查

pom.xml 结构

Maven 配置文件，相当于 Java 世界的 package.json：

  • <dependencies> — 项目依赖列表
  • <properties> — 定义变量（如版本号），用 ${变量名} 引用
  • <dependencyManagement> — 统一管理依赖版本，子模块不用再写版本号
  • <build> — 构建配置，如打包插件

@SpringBootApplication

组合注解，包含三个注解：

  • @Configuration — 标记为配置类
  • @EnableAutoConfiguration — 开启自动配置（根据 classpath 的依赖自动创建 Bean）
  • @ComponentScan — 扫描当前包及其子包，找到所有 @Controller、@Service、@Component 等

Starter 机制

Spring Boot 的"一键引入"。例如引入 spring-boot-starter-web：

  • 自动引入 Tomcat（Web 服务器）
  • 自动引入 Spring MVC（Web 框架）
  • 自动引入 Jackson（JSON 处理）
  • 自动配置上述组件（无需写配置类）

统一返回格式

后续要写的 Result.java 会让所有 API 返回统一的 JSON 结构：

{
  "code": 200,      // 状态码（200成功，其他失败）
  "message": "ok",  // 提示信息
  "data": { ... }   // 实际数据
}

这样做的好处：前端只用写一套解析逻辑，不用每次猜后端返回的格式。
