什么是 Maven 父工程？

就像 npm workspace 的根 package.json，定义一个容器项目，不写代码，只声明有哪些子模块。后期可加更多子模块，一条 mvn 命令编译全部。

<groupId>com.testplatform</groupId>          <!-- 组织ID -->
<artifactId>test-platform-parent</artifactId> <!-- 项目名 -->
<version>1.0.0</version>                    <!-- 版本 -->
<packaging>pom</packaging>                  <!-- 父容器类型 -->
<modules><module>backend</module></modules> <!-- 子模块 -->

逐行拆解

代码解释
package com.testplatform包声明，对应 java/com/testplatform/ 目录
@SpringBootApplication组合注解 = @Configuration + @EnableAutoConfiguration + @ComponentScan
@Configuration标记为配置类，可定义 Bean
@EnableAutoConfiguration自动配置——根据 classpath jar 自动猜你要做什么
@ComponentScan扫描 com.testplatform 包下所有 @Controller、@Service 等并注册为 Bean
main 方法Java 程序入口
SpringApplication.run()创建 Spring 容器 → 加载配置 → 自动配置 → 启动 Tomcat

启动流程

main() 执行
  |-> SpringApplication.run() 被调用
  |-> 确定应用类型（Web 应用）
  |-> 加载 spring.factories（自动配置菜单）
  |-> 读取 application.yml
  |-> 创建 Spring 容器
  |     |-> 扫描所有 @Component、@Service、@Controller

  |     |-> 注册所有 Bean
  |-> 启动内嵌 Tomcat（默认 8080 端口）
  |-> 输出 "Started TestPlatformApplication"

3. 核心概念详解

Spring Boot 是什么？

Java Web 框架，最大特点是"自动配置"：

  • Spring Boot : Java = NestJS : Node.js = FastAPI : Python
  • 内嵌 Tomcat，无需单独装 Web 服务器
  • Starter 一键引入整组依赖

Maven 是什么？

Java 世界的包管理器：

Mavennpm 类比
pom.xmlpackage.json
Maven Centralnpm registry
mvn clean installnpm run build
~/.m2/repositorynode_modules

MyBatis-Plus 是什么？

ORM 框架，不写 SQL 就能 CRUD：

userMapper.insert(user);         // 新增
userMapper.selectById(1L);       // 按 ID 查
userMapper.updateById(user);     // 修改
userMapper.selectList(null);     // 查全部

比 JPA 灵活，比原生 MyBatis 高效。

三层架构

Controller → 接收请求、返回结果
Service    → 业务逻辑（if/else/计算）
Mapper     → 数据库操作（CRUD）

优点：解耦、复用、可测试

3.1 IoC 和 DI（Spring 核心概念）

概念解释类比
IoC 控制反转不自己 new 对象，交给 Spring 容器创建不自己做饭，告诉服务员要什么
DI 依赖注入Spring 自动把需要的对象注入进来服务员直接把菜端到你面前
// 传统写法（自己 new，耦合度高）
public class UserController {
    private UserService userService = new UserServiceImpl();
}

// Spring 写法（IoC + DI，低耦合）
@RestController
public class UserController {
    @Autowired
    private UserService userService;  // Spring 自动赋值，不用 new
}

3.2 执行流程（V1 核心逻辑）

1. 用户在页面创建测试用例（填写 URL、方法、参数、预期结果）
2. 在用例列表点击「执行」按钮
3. 后端接收请求 → 用 RestTemplate 发送真实 HTTP 请求
4. 比对实际响应与预期结果 → 生成 PASS / FAIL / ERROR
5. 保存 execution_record（含请求详情、响应详情）
6. 返回执行结果给前端
7. 用户可在执行记录页查看所有历史日志

3.3 统一返回格式 Result<T>

{
  "code": 200,        // 200=成功，其他=失败
  "message": "ok",    // 提示信息
  "data": { ... }     // 实际数据
}

所有 API 接口返回同样的 JSON 结构，前端只用写一套解析逻辑。

// 成功
return Result.success(data);
return Result.success("查询成功", data);

// 失败
return Result.error("用户名不能为空");
return Result.error(401, "Token 已过期");

5.2 建库建表语句

CREATE DATABASE IF NOT EXISTS test_platform
    DEFAULT CHARACTER SET utf8mb4
    DEFAULT COLLATE utf8mb4_general_ci;

USE test_platform;

CREATE TABLE test_case (
    id              BIGINT(20)   NOT NULL AUTO_INCREMENT,
    name            VARCHAR(255) NOT NULL,
    request_url     VARCHAR(1024) NOT NULL,
    request_method  VARCHAR(10)   NOT NULL DEFAULT 'GET',
    request_headers TEXT,
    request_params  TEXT,
    expected_result TEXT,
    create_time     DATETIME     DEFAULT CURRENT_TIMESTAMP,
    update_time     DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

CREATE TABLE execution_record (
    id              BIGINT(20)   NOT NULL AUTO_INCREMENT,
    test_case_id    BIGINT(20)   NOT NULL,
    status          VARCHAR(10) NOT NULL DEFAULT 'PENDING',
    request_detail  TEXT,
    response_detail TEXT,
    actual_result   TEXT,
    execute_time    DATETIME     DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (id),
    KEY idx_test_case_id (test_case_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
